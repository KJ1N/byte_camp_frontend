# 评估与交付方案

## 1. 交付物

| 交付物 | 要求 |
| --- | --- |
| 可运行系统 | 线上 URL 或二维码 |
| GitHub 代码 | 提交到 main/master 分支 |
| PRD | 产品目标、用户流程、功能范围、验收标准 |
| 技术架构 | 模块边界、数据模型、AI/审核/榜单设计 |
| 审核规则库 | 风险分类、处理策略、质量评分标准 |
| 效果评估报告 | 生成、审核、分发和性能评估 |

## 2. 评估指标

| 维度 | 指标 |
| --- | --- |
| AI 应用 | 生成成功率、改写采纳率、审核可解释性 |
| 产品体验 | 创作到发布链路是否顺畅 |
| 安全质量 | 高危内容识别率、误杀率、质量分稳定性 |
| 性能 | 首页 LCP、榜单加载耗时 |
| 工程化 | 测试、CI、代码结构、文档完整度 |

## 3. 演示脚本

1. 注册并登录。
2. 进入首页内容信息流，确认首页展示推荐内容、热点榜和爆文榜入口。
3. hover 右上角用户名称，展示“工作台”“草稿箱”“退出登录”菜单。
4. 进入工作台，输入主题：“AI 如何改变内容创作”。
5. 选择风格：“科普/头条资讯”。
6. 生成标题、大纲和正文。
7. 保存为草稿并进入编辑器。
8. 在编辑器中修改一段内容。
9. 展示自动保存和版本记录。
10. 插入一段风险内容，触发审核。
11. 使用一键合规改写。
12. 发布文章。
13. 在榜单和详情页查看内容。
14. 展示质量分、审核记录和阅读数据。
15. 回到首页 hover 用户名称，点击“退出登录”，确认登录态被清理。

## 4. 风险与应对

| 风险 | 应对 |
| --- | --- |
| AI API 不稳定 | 增加重试、超时和 mock 模式 |
| 审核误判 | 给出证据片段，允许中低风险重审 |
| 时间不足 | 优先保证端到端闭环 |
| 榜单数据不足 | 使用 seed 数据和模拟互动事件 |
| 部署复杂 | 使用托管数据库和平台部署 |
