# AI 创作者辅助生产与分发平台 PRD

## 1. 文档信息

| 项目 | 内容 |
| --- | --- |
| 产品名称 | AI Creator Hub |
| 产品定位 | 面向图文创作者的 AI 辅助生产、审核与分发平台 |
| 版本 | v1.0 |
| 目标周期 | 3 周训练营交付 |
| 目标形态 | 可线上访问、可演示、可评估、可扩展的全栈系统 |

## 2. 背景与问题

内容创作者正在越来越多地使用 AIGC 工具完成选题、标题、正文、配图和改写。但现有工具往往停留在“单点生成”：生成一段文本、改一段标题、给一张图片。真实内容生产需要的是一条完整链路：

```text
灵感输入 -> AI 生成 -> 人工编辑 -> 草稿保存 -> 内容审核 -> 质量打分 -> 发布分发 -> 数据反馈 -> 二次优化
```

平台需要同时扮演三个角色：

- 得力助手：降低创作门槛，帮助用户快速生成、编辑和优化内容。
- 守门员：在创作和发布阶段发现安全风险、低质内容和敏感信息。
- 导航员：把优质内容通过榜单和信息流分发出去，并把消费数据反馈给创作者。

## 3. 产品目标

### 3.1 业务目标

- 建立从创作到分发的端到端闭环。
- 让创作者可以在 5 分钟内完成一篇可发布的中长图文草稿。
- 发布前必须经过 AI 安全审核和质量评分。
- 支持内容榜单，帮助读者发现优质内容。
- 形成可解释的内容质量体系和审核规则库，便于答辩与后续优化。

### 3.2 工程目标

- 使用现代全栈技术搭建可维护 monorepo。
- 前后端职责清晰，AI、审核、评分逻辑集中在后端。
- 数据模型支持草稿、版本、发布快照、评分、审核记录和阅读反馈。
- 核心页面首屏 LCP 目标不超过 2.5 秒。
- 具备基础单元测试、接口测试和端到端测试。

## 4. 用户画像

| 用户 | 场景 | 核心诉求 |
| --- | --- | --- |
| 新手创作者 | 只有一个主题或关键词 | 快速得到结构完整的文章初稿 |
| 熟练创作者 | 已有素材和表达风格 | 让 AI 辅助改写、扩写、标题优化 |
| 内容运营 | 需要管理质量和热点 | 查看榜单、质量分和内容风险 |
| 审核人员 | 关注违规内容 | 拦截高危内容，追踪审核原因 |
| 读者 | 浏览优质图文 | 快速看到热点和爆文内容 |

## 5. 用户流程

### 5.1 创作者主流程

1. 用户注册/登录。
2. 进入创作工作台。
3. 输入主题、关键词、目标受众、内容风格。
4. 选择 Prompt 模板或使用默认模板。
5. AI 生成标题、大纲和正文。
6. 用户在富文本编辑器中修改内容。
7. 系统每 30 秒自动保存草稿；断网时本地暂存，恢复网络后同步。
8. 用户点击发布。
9. 系统执行发布前 AI 审核和质量评分。
10. 高危内容被拦截；中低风险内容给出修改建议。
11. 审核通过后发布，生成内容详情页。
12. 内容进入信息流和榜单。
13. 用户查看阅读、点赞、收藏、质量反馈，并进行二次优化。

### 5.2 读者主流程

1. 进入首页。
2. 浏览推荐流、热点榜、爆文榜。
3. 无限滚动加载更多内容。
4. 打开内容详情页。
5. 触发阅读、点赞、收藏等反馈数据。

### 6.4 内容 AI 审核

| --- | --- | --- |

| 发布后巡检 | P2 | 定时复查和举报复审 |

### 6.5 内容质量评分

评分维度如下：

| 维度 | 权重 | 说明 |
| --- | --- | --- |
| 内容价值 | 30% | 信息量、观点深度、实用性 |
| 表达质量 | 25% | 结构清晰、语言流畅、逻辑完整 |
| 读者体验 | 20% | 可读性、节奏、标题吸引力 |
| 传播潜力 | 15% | 热点相关性、话题性、互动潜力 |
| 合规安全 | 10% | 风险等级、是否可发布 |

综合分公式：

```text
quality_score =
  content_value * 0.30 +
  expression_quality * 0.25 +
  reader_experience * 0.20 +
  spread_potential * 0.15 +
  safety_score * 0.10
```

### 6.6 发布与分发

| 功能 | 优先级 | 说明 |
| --- | --- | --- |
| 内容发布 | P0 | 审核通过后生成发布快照 |
| 内容详情 | P0 | 展示作者、时间、正文、互动数据 |
| 内容撤回 | P1 | 作者可以撤回已发布内容 |
| 内容更新 | P0 | 二次编辑后重新审核并发布新版本 |
| 推荐信息流 | P0 | 按综合排序展示 |
| 热点榜 | P0 | 结合阅读热度和时间衰减 |
| 爆文榜 | P0 | 结合质量分、热度、互动数据 |
| 无限滚动 | P0 | 榜单和信息流支持分页加载 |

榜单排序公式：

```text
rank_score =
  quality_score * 0.45 +
  hot_score * 0.35 +
  freshness_score * 0.15 +
  feedback_score * 0.05
```

## 7. MVP 边界

### 7.1 必做范围

- 用户注册、登录、退出。
- AI 生成标题、大纲、正文。
- 富文本编辑器。
- 草稿自动保存。
- 发布前审核。
- 质量评分。
- 发布内容详情。
- 热点榜/爆文榜基础排序。
- 无限滚动。
- 技术文档和评估报告。

### 7.2 暂不做或后置

- 真实接入头条/抖音开放 API。
- 复杂个性化推荐。
- 多人协作编辑。
- 大规模发布后巡检。
- 多模态视频生成。
- 完整后台运营系统。

## 8. 非功能需求

| 类别 | 要求 |
| --- | --- |
| 性能 | 首页和榜单 LCP ≤ 2.5 秒 |
| 可用性 | AI 调用失败时提供重试和降级提示 |
| 安全 | 密码加密存储，JWT 过期控制 |
| 合规 | 发布前必须审核；审核记录可追溯 |
| 可维护 | 模块边界清晰，业务逻辑不散落在前端 |
| 可测试 | 关键服务有单测，核心流程有 e2e |
| 可观测 | 记录 AI 请求、审核结果、发布状态和榜单刷新日志 |

## 9. 里程碑

| 周期 | 目标 | 交付 |
| --- | --- | --- |
| 第 1 周 | 工程基础与创作闭环 | monorepo、登录、编辑器、AI 生成 |
| 第 2 周 | 审核与评分闭环 | 安全审核、质量评分、发布流程 |
| 第 3 周 | 分发与交付 | 榜单、性能优化、部署、文档、答辩材料 |

## 10. 验收标准

- 用户可以完整完成登录、创作、编辑、审核、发布、浏览榜单流程。
- 发布动作必须经过 AI 审核和质量评分。
- 违规内容会被拦截或要求修改。
- 榜单可以滚动加载，排序结果可解释。
- 文档包含 PRD、架构、审核规则、评估方案。
- 代码提交到 GitHub 主分支。
- 线上服务可公开访问。
