# 内容安全审核规则与质量评估体系

## 1. 审核目标

审核系统负责在内容发布前识别明显违规、高风险和低质量内容，并给出可解释的处理结果。系统不是简单地“拒绝”，而是尽量形成：

```text
识别风险 -> 给出原因 -> 提供修改建议 -> 重新审核 -> 安全发布
```

## 2. 风险分类

| 风险 | 等级 | 处理 |
| --- | --- | --- |
| 涉黄低俗 | BLOCK/WARN | 高危拦截，低俗表达建议改写 |
| 赌博引导 | BLOCK | 禁止发布 |
| 毒品/违禁品 | BLOCK | 禁止发布 |
| 敏感个人信息 | BLOCK/WARN | 脱敏或删除 |
| 违法犯罪引导 | BLOCK | 禁止发布 |
| 虚假医疗/金融建议 | WARN/BLOCK | 提示风险，严重时拦截 |
| 广告导流 | WARN | 提醒平台规则 |
| 事实不确定 | WARN | 要求补充来源或降低绝对化表达 |
| 低质量内容 | WARN | 给出优化建议 |

## 3. 审核输出结构

```json
{
  "decision": "PASS | WARN | BLOCK",
  "riskLevel": "none | low | medium | high",
  "categories": ["sensitive_info"],
  "evidence": [
    {
      "text": "命中的片段",
      "reason": "为什么有风险"
    }
  ],
  "rewriteSuggestions": ["替代表达"],
  "summary": "整体审核说明"
}
```

## 4. 质量评分标准

| 维度 | 高分标准 | 低分表现 |
| --- | --- | --- |
| 内容价值 | 有信息密度、观点或实用建议 | 空泛、重复、缺少结论 |
| 表达质量 | 结构清楚、语言自然、逻辑顺 | 跳跃、冗长、语病明显 |
| 读者体验 | 标题明确、段落节奏好 | 标题含糊、段落过长 |
| 传播潜力 | 贴近热点、有讨论空间 | 与用户兴趣弱相关 |
| 合规安全 | 无明显风险 | 存在敏感或误导表达 |

## 5. 人机协同机制

- AI 给出判断和建议。
- 用户确认是否采纳改写。
- 高危风险不能绕过。
- 中低风险允许修改后重新提交。
- 每次审核结果写入 `audit_records`，用于后续评估。
